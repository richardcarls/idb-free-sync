# Security Policy

## Reporting a Vulnerability

Do not open a public issue for a suspected vulnerability.

Use GitHub's private vulnerability reporting or security advisory feature for
this repository. Include reproduction steps, affected versions, impact, and
any suggested mitigation.

Please allow maintainers time to investigate before publicly disclosing the
issue. You can expect acknowledgment after the report is received and updates
as the investigation progresses.

## Supported Versions

Security fixes are applied to the latest released version.

Never include OAuth tokens, passwords, client secrets, or private synced data
in reports, issues, or pull requests.
